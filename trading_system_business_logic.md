# 📊 Бизнес-логика торговой системы Trading Analyzer v6.0

## 🎯 ЧТО ДЕЛАЕТ СИСТЕМА

Система анализирует **ценовые диапазоны** и **торговые сигналы** на основе концепции **БЛОКОВ** и **СЕССИЙ**.

---

## 🏗️ ОСНОВНАЯ КОНЦЕПЦИЯ: БЛОКИ vs СЕССИИ

### 📦 БЛОКИ (Формирование диапазонов)
- **Время**: например, 00:00 - 13:00 UTC
- **Задача**: определить границы ценового диапазона (High/Low)
- **Результат**: получаем **range_high** и **range_low** за этот период

**Пример БЛОКИ:**
```
БЛОК 00:00-13:00:
- Минимальная цена: 23320.25 (range_low)
- Максимальная цена: 23420.50 (range_high)
- Размер диапазона: 100.25 пунктов
```




### 🎯 СЕССИИ (Анализ торговли)
- **Время**: например, 14:00 - 20:00 UTC  
- **Задача**: анализировать как цена взаимодействует с границами БЛОКА
- **Результат**: определить тип входа в позицию

**Пример СЕССИИ:**
```
СЕССИЯ 14:00-20:00:
- Анализируем поведение цены относительно границ 23320.25-23420.50
- Фиксируем: пробои, возвраты, лимитные касания
```

---

**КОСНУЛАСЬ** или **КАСАНИЕ**
candle['low'] <= range_low <= candle['high']
candle['low'] <= range_high <= candle['high']
___

## 🚪 ТИПЫ ВХОДОВ В ПОЗИЦИЮ ДЛЯ РЕЖИМА ПО-ТРЕНДУ

### 1. **ENTRY_LONG_TREND** (маркетный вход LONG)
- **Условие**: цена на старте СЕССИИ находится В диапазона БЛОКА и **КОСНУЛАСЬ** range_high
- **Логика**: покупаем при касании верхней границы
- **Пример**: цена была 23410.50, коснулась 23420.50 → ENTRY_LONG по 23420.50

### 2. **ENTRY_SHORT_TREND** (маркетный вход SHORT)
- **Условие**: цена на старте СЕССИИ находится В диапазона БЛОКА **КОСНУЛАСЬ** range_low
- **Логика**: продаем при касании нижней границы
- **Пример**: цена была 23330.25, коснулась 23320.25 → ENTRY_SHORT по 23320.25

### 3. **LIMIT_LONG_TREND** (лимитный вход LONG)
- **Условие**: цена на старте СЕССИИ находится ВЫШЕ диапазона БЛОКА во время СЕССИИ **КОСНУЛАСЬ** range_high 
- **Логика**: покупаем от верхней границы БЛОКА
- **Пример**: цена была 23450.50, коснулась 23420.50 → LIMIT_LONG по 23420.50

### 4. **LIMIT_SHORT_TREND** (лимитный вход SHORT)
- **Условие**: цена на старте СЕССИИ находится НИЖЕ диапазона БЛОКА во время СЕССИИ **КОСНУЛАСЬ** range_low 
- **Логика**: продаем от нижней границы БЛОКА
- **Пример**: цена была 23300.25, коснулась 23320.25 → LIMIT_SHORT по 23320.25

### 5. **INSIDE_BLOCK** (ожидание пробоя)
- **Условие**: нет подходящих условий для входа. Цена всё время СЕССИИ находится В диапазона БЛОКА
- **Логика**: система не дает сигналов

### 6. **OUTSIDE_BLOCK** (ожидание возврата)
- **Условие**: нет подходящих условий для входа. Цена всё время СЕССИИ находится ВНЕ диапазона БЛОКА
- **Логика**: система не дает сигналов

## ⚙️ РЕЖИМЫ РАБОТЫ СИСТЕМЫ

### 🎯 РЕЖИМ ПО-ТРЕНДУ (use_return_mode = False)
**Логика:**
1. На старте СЕССИИ проверяем позицию цены
2. Если цена **ВНУТРИ диапазона БЛОКА** → ищем ENTRY_LONG_TREND/ENTRY_SHORT_TREND
3. Если цена **ВНЕ диапазона БЛОКа** → OUTSIDE_BLOCK, затем ищем LIMIT_LONG_TREND/LIMIT_SHORT_TREND

**Пример:**
```
БЛОК создал диапазон: 23320.25-23420.50
СЕССИЯ началась с ценой 23450.50 (ВЫШЕ диапазона)
Ждем возврата: цена упала до 23420.50 (коснулась границы)
Результат: LIMIT_LONG_TREND по 23420.50
```

### 🔄 РЕЖИМ ВОЗВРАТНЫЙ (use_return_mode = True)
**Логика:**
1. На старте СЕССИИ проверяем позицию цены
2. Если цена **ВНУТРИ диапазона БЛОКА** → ищем ENTRY_LONG_REVERSE/ENTRY_SHORT_REVERSE
3. Если цена **ВНЕ диапазона БЛОКа** → OUTSIDE_BLOCK, затем ищем LIMIT_LONG_REVERSE/LIMIT_SHORT_REVERSE

**Пример:**
```
БЛОК создал диапазон: 23320.25-23420.50
СЕССИЯ началась с ценой 23450.50 (ВЫШЕ диапазона)
Ждем возврата: цена упала до 23420.50 (коснулась границы)
Результат: LIMIT_SHORT_REVERSE по 23420.50
```

## 📈 АНАЛИЗ РЕЗУЛЬТАТОВ СДЕЛОК

### TP/SL Логика:
После входа в позицию рассчитываем:
- **Take Profit** = entry_price ± (range_size × tp_multiplier)
- **Stop Loss** = entry_price ± (range_size × sl_multiplier)

### Результаты сделок:
- **TP** - Take Profit достигнут
- **SL** - Stop Loss достигнут  
- **BE** - Break Even (закрытие по безубытку)
- **SESSION_CLOSE** - принудительное закрытие в конце сессии

---

## 📰 ФИЛЬТРАЦИЯ ПО НОВОСТЯМ

### Календарь новостей:
- **High Impact** - важные события (например, Non-Farm Payrolls)
- **Medium Impact** - средней важности
- **Buffer Time** - буферное время до/после новости

### Логика блокировки:
1. Проверяем календарь перед входом в позицию
2. Если важная новость в течение buffer_minutes → **блокируем вход**
3. Если позиция уже открыта и приближается новость → **принудительное закрытие**

## 🔢 R-СИСТЕМА РАСЧЕТОВ

### R-единица:
- **1R** = размер одного Stop Loss
- **R-результат** = actual_pnl ÷ sl_size

### Примеры:
```
Сделка ENTRY_LONG_TREND:
- Вход: 23320.25
- SL: 23290.25 (30 пунктов = 1R)
- Выход: 23350.25 (прибыль 30 пунктов)
- R-результат: +1.0R

Сделка LIMIT_SHORT_TREND:
- Вход: 23420.50  
- SL: 23450.50 (30 пунктов = 1R)
- Выход: 23390.50 (прибыль 30 пунктов)
- R-результат: +1.0R

## 📊 СТРУКТУРА РЕЗУЛЬТАТОВ

### Каждый день анализа содержит:
```python
{
    'date': datetime,           # Дата анализа
    'range_high': float,        # Верхняя граница БЛОКА
    'range_low': float,         # Нижняя граница БЛОКА  
    'range_size': float,        # Размер диапазона в пунктах
    'entry_type': str,          # Тип входа (ENTRY_LONG_TREND, LIMIT_SHORT_REVERSE, etc.)
    'entry_price': float,       # Цена входа в позицию
    'entry_time': datetime,     # Время входа в позицию
    'exit_price': float,        # Цена выхода 
    'exit_time': datetime,      # Время выхода из позиции
    'result': str,              # Результат (TP/SL/BE/SESSION_CLOSE)
    'r_result': float,          # R-результат сделки
    'close_reason': str,        # Причина закрытия
}
```

## 🎯 ЦЕЛЬ СИСТЕМЫ

**Основная задача**: найти **оптимальные параметры** торговой стратегии:
- Какое время лучше для БЛОКОВ?
- Какое время лучше для СЕССИЙ?
- Какие множители TP/SL дают лучший R-результат?
- Какой режим эффективнее: строгий или возвратный?

**R-ориентированность**: система фокусируется на **скорости достижения** целевого R-результата (например, +5R), а не на общей прибыли.
