# Trading Analyzer v6.0

Система анализа торговых стратегий на основе ценовых диапазонов (БЛОКОВ) и торговых сессий.

## 🚀 Основные возможности

- Анализ торговых стратегий на основе БЛОКОВ и СЕССИЙ
- Два режима работы: по-тренду и возвратный
- Гибкая настройка параметров TP/SL
- Фильтрация по новостям
- Оптимизация параметров
- Детальные отчеты и визуализация
- R-метрика для оценки эффективности

## 📋 Требования

- Python 3.8+
- Streamlit
- Pandas
- Plotly
- NumPy

## 🛠 Установка

1. Клонируйте репозиторий:
```bash
git clone https://github.com/yourusername/trading-analyzer.git
cd trading-analyzer
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv_trading
source venv_trading/bin/activate  # Linux/Mac
# или
venv_trading\Scripts\activate  # Windows
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

## 🏃‍♂️ Запуск

```bash
streamlit run app.py
```

Или используйте скрипт запуска:
```bash
./run_trading.sh  # Linux/Mac
```

## 📊 Формат входных данных

### Ценовые данные (CSV)
Обязательные колонки:
- `timestamp` - временная метка (формат: YYYY-MM-DD HH:MM:SS)
- `open` - цена открытия
- `high` - максимум свечи
- `low` - минимум свечи
- `close` - цена закрытия

### Новостные данные (CSV) - опционально
Обязательные колонки:
- `timestamp` - время новости
- `impact` - важность (high/medium/low)
- `Currency` или `currency` - валюта (EUR, USD, GBP и т.д.)

## 🎯 Основные концепции

### БЛОК
Временной период формирования ценового диапазона. Определяет границы (high/low) для последующей торговли.

### СЕССИЯ
Торговый период, в течение которого происходит поиск входов и исполнение сделок.

### Типы входов
- **ENTRY_LONG_TREND** - вход в лонг при пробое верхней границы (режим по-тренду)
- **ENTRY_SHORT_TREND** - вход в шорт при пробое нижней границы (режим по-тренду)
- **LIMIT_LONG_TREND** - лимитный вход в лонг (режим по-тренду)
- **LIMIT_SHORT_TREND** - лимитный вход в шорт (режим по-тренду)
- **ENTRY_LONG_REVERSE** - вход в лонг от нижней границы (возвратный режим)
- **ENTRY_SHORT_REVERSE** - вход в шорт от верхней границы (возвратный режим)
- **LIMIT_LONG_REVERSE** - лимитный вход в лонг (возвратный режим)
- **LIMIT_SHORT_REVERSE** - лимитный вход в шорт (возвратный режим)

## ⚙️ Настройки стратегии

### Временные настройки
- **Начало/Конец БЛОКА** - время формирования ценового диапазона
- **Начало/Конец СЕССИИ** - время торговой сессии
- **Начать БЛОК с предыдущего дня** 🆕 - формировать БЛОК начиная с предыдущего торгового дня

### Режимы работы
- **По-тренду** - торговля в направлении пробоя границ
- **Возвратный** - торговля от границ внутрь диапазона

### Параметры TP/SL
- **Множители TP/SL** - коэффициенты от размера диапазона
- **Коэффициент R для TP** - корректировка для учета комиссий
- **Фиксированные TP/SL** - использование фиксированных значений при выходе диапазона за пороги

### Фильтры
- **Размер диапазона** - минимальный и максимальный размер БЛОКА
- **Торговые дни** - выбор дней недели для анализа
- **Новостной фильтр** - блокировка входов около важных новостей
- **Пропуск дней с красными новостями** 🆕 - полное исключение дней с high impact новостями

## 📈 Метрики эффективности

### R-метрика
Универсальная метрика оценки сделок:
- **R = 1.0** - прибыль равна риску
- **R > 1.0** - прибыль превышает риск
- **R < 0** - убыточная сделка

### Статистика
- **Total R** - суммарный результат в R
- **Win Rate** - процент прибыльных сделок
- **Average R per trade** - средний R на сделку
- **Profit Factor** - отношение прибыли к убыткам

## 🔄 Оптимизация

Система поддерживает автоматическую оптимизацию параметров TP/SL:
1. Выберите диапазоны TP и SL множителей
2. Укажите целевую метрику (максимальный R, win rate и т.д.)
3. Запустите оптимизацию
4. Получите тепловую карту и топ-10 комбинаций

## 🆕 Новые функции v6.0

### 📅 БЛОК с предыдущего дня
Позволяет формировать ценовой диапазон, начиная с указанного времени предыдущего торгового дня. Полезно для стратегий, использующих азиатскую или американскую сессию предыдущего дня.

**Как использовать:**
1. Активируйте checkbox "📅 Начать БЛОК с предыдущего дня"
2. Укажите время начала БЛОКА
3. БЛОК будет формироваться с этого времени предыдущего дня

### 🚫 Пропуск дней с красными новостями
Автоматически исключает из анализа торговые дни, в которых есть новости высокой важности (high impact).

**Как использовать:**
1. Включите новостной фильтр
2. Активируйте "🚫 Пропускать дни с красными новостями"
3. Выберите валюты для отслеживания
4. Дни с красными новостями будут полностью пропущены

## 🐛 Известные особенности

- При использовании БЛОКА с предыдущего дня убедитесь, что данные за предыдущий день доступны
- Новостной фильтр работает только при наличии загруженных новостных данных
- Временные зоны должны быть согласованы между ценовыми и новостными данными
