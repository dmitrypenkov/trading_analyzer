## 📂 Загрузка данных

### Формат ценовых данных

CSV файл должен содержать следующие колонки:
```csv
timestamp,open,high,low,close
2024-01-01 00:00:00,1234.56,1235.00,1234.00,1234.75
2024-01-01 01:00:00,1234.75,1235.25,1234.50,1235.00
```

**Важно:**
- `timestamp` - формат YYYY-MM-DD HH:MM:SS
- Все цены должны быть числовыми
- Данные должны быть отсортированы по времени

### Формат новостных данных (опционально)

```csv
timestamp,impact,Currency,event
2024-01-01 14:30:00,high,USD,Non-Farm Payrolls
2024-01-01 15:00:00,medium,EUR,ECB Speech
```

**Поля:**
- `impact` - важность: high (красная), medium (оранжевая), low (желтая)
- `Currency` - код валюты: USD, EUR, GBP и т.д.
- `event` - название события (опционально)

## ⚙️ Настройка стратегии

### Вкладка "📊 Основные"

#### Настройки БЛОКА
- **Начало БЛОКА** - время начала формирования диапазона (UTC)
- **Конец БЛОКА** - время окончания формирования диапазона (UTC)
- **📅 Начать БЛОК с предыдущего дня** - если активно, БЛОК начинается с предыдущего дня

**Пример:** БЛОК 00:00-13:00 формирует диапазон азиатской и европейской сессии

#### Настройки СЕССИИ
- **Начало СЕССИИ** - время начала торговли (UTC)
- **Конец СЕССИИ** - время окончания торговли (UTC)

**Пример:** СЕССИЯ 14:00-20:00 для торговли в американскую сессию

#### Режим работы
- **🎯 По-тренду** - вход в направлении пробоя границ
- **🔄 Возвратный** - вход от границ внутрь диапазона

#### Фильтры размера диапазона
- **Минимальный размер** - пропуск дней с маленьким диапазоном
- **Максимальный размер** - пропуск дней с большим диапазоном

### Вкладка "🎯 TP/SL"

#### Стандартные настройки
- **TP множитель** - Take Profit = размер_диапазона × множитель
- **SL множитель** - Stop Loss = размер_диапазона × множитель
- **Коэффициент R для TP** - корректировка для учета комиссий (0.9 = -10% от TP)

#### Фиксированные TP/SL
Используются когда диапазон выходит за установленные пороги:
- **Минимальный порог** - если диапазон меньше, используем фиксированные значения
- **Максимальный порог** - если диапазон больше, используем фиксированные значения
- **Фиксированное расстояние до TP** - в пунктах
- **Фиксированное расстояние до SL** - в пунктах

### Вкладка "📰 Новости"

#### Фильтр новостей для входов
- **Важность новостей** - какие новости блокируют входы (high/medium/low)
- **Валюты для фильтрации** - отслеживать новости только по выбранным валютам
- **Буферное время** - блокировка входов за X минут до/после новости

#### Пропуск дней с красными новостями
- **🚫 Пропускать дни с красными новостями** - полностью исключить дни с high impact новостями
- Использует те же валюты из фильтра

### Вкладка "💾 Сохранение"

#### Сохранение настроек
1. Введите название пресета
2. Нажмите "💾 Сохранить настройки"
3. Скачайте JSON файл для использования в будущем

#### Загрузка настроек
1. Нажмите на область загрузки файлов
2. Выберите сохраненный JSON файл
3. Нажмите "📥 Применить загруженные настройки"

## 📈 Запуск анализа

### Основной анализ
1. Убедитесь, что данные загружены
2. Проверьте настройки
3. Нажмите "🚀 Запустить анализ"
4. Дождитесь завершения

### Что происходит при анализе:
1. **Фильтрация дней** - отбор торговых дней по настройкам
2. **Формирование БЛОКА** - расчет границ для каждого дня
3. **Поиск входов** - определение типа входа в СЕССИИ
4. **Симуляция сделок** - расчет TP/SL и результата
5. **Расчет метрик** - вычисление R-результата и статистики

## 📊 Работа с результатами

### Уровни просмотра

#### Общий обзор
- Суммарная статистика за весь период
- Накопительный график R-результата
- Основные метрики эффективности

#### Годовые отчеты
- Статистика по каждому году
- График накопительного R за год
- Таблица месячных результатов

#### Месячные отчеты
- Детальная таблица всех сделок
- Фильтрация по различным параметрам
- Экспорт в CSV

### Фильтрация сделок
- **Результат** - TP/SL/BE/NO_TRADE
- **Тип входа** - выбор конкретных типов
- **R-диапазон** - фильтр по результату в R
- **Дни недели** - только определенные дни
- **Блокированные новостями** - показать/скрыть

## 🔄 Оптимизация параметров

### Настройка оптимизации

1. Перейдите в раздел "🎛️ Оптимизация"
2. Установите диапазоны для перебора:
   - **TP диапазон** - от/до/шаг для TP множителя
   - **SL диапазон** - от/до/шаг для SL множителя

3. Выберите целевую метрику:
   - **Максимальный Total R** - общая прибыльность
   - **Максимальный Win Rate** - процент выигрышей
   - **Лучший R-Ratio** - соотношение риск/прибыль

4. Опции:
   - **Параллельная обработка** - ускорение за счет многопоточности
   - **Использовать кеш** - повторное использование расчетов

### Запуск оптимизации

1. Нажмите "🚀 Запустить оптимизацию"
2. Следите за прогресс-баром
3. Дождитесь завершения

### Анализ результатов

#### Тепловая карта
- Визуализация всех комбинаций
- Цвет показывает значение целевой метрики
- Наведите для просмотра деталей

#### Топ-10 комбинаций

- Сортировка по целевой метрике
- Все ключевые показатели

#### Применение результатов
1. Выберите оптимальную комбинацию
2. Вернитесь в настройки
3. Установите найденные значения TP/SL

## ❓ Часто задаваемые вопросы

### В: Какой timeframe использовать?
**О:** Система работает с любым timeframe. Рекомендуется H1 для внутридневной торговли.

### В: Почему некоторые дни пропускаются?
**О:** Дни могут пропускаться по причинам:
- Нет данных для БЛОКА или СЕССИИ
- Размер диапазона вне установленных фильтров
- Наличие красных новостей (если включен фильтр)
- День недели не выбран в настройках

### В: Что такое R-метрика?
**О:** R - это отношение прибыли/убытка к риску:
- R = 1.0 означает, что прибыль равна риску
- R = 2.0 означает, что прибыль в 2 раза больше риска
- R = -1.0 означает убыток равный риску

### В: Как работает БЛОК с предыдущего дня?
**О:** При активации этой опции:
1. БЛОК начинается с указанного времени предыдущего дня
2. Заканчивается в указанное время текущего дня
3. Полезно для захвата ночной сессии

### В: Что делает новостной фильтр?
**О:** Два уровня фильтрации:
1. **Блокировка входов** - не открывает сделки около новостей
2. **Пропуск дней** - полностью исключает дни с красными новостями

### В: Можно ли использовать без новостей?
**О:** Да, новостные данные опциональны. Просто не активируйте новостной фильтр.

### В: Как сохранить результаты?
**О:** Несколько способов:
- Экспорт сделок в CSV
- Сохранение настроек в JSON
- Скриншоты графиков
- Копирование таблиц

### В: Почему анализ идет долго?
**О:** Скорость зависит от:
- Объема данных
- Периода анализа
- Количества торговых дней
- Мощности компьютера

Для ускорения:
- Уменьшите период анализа
- Используйте фильтры дней
- При оптимизации включите параллельную обработку
