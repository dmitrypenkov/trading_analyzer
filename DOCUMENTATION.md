# Trading Analyzer v11.0 — Документация

## Что это

Система бэктестинга торговых стратегий на основе ценовых диапазонов (Range Breakout / Range Reversion). Написана на Python + Streamlit. Анализирует исторические OHLC-данные, определяет входы от границ диапазона, симулирует исполнение сделок и измеряет результаты в R-единицах.

Применяется к любому инструменту: фьючерсы на криптовалюты, Forex, индексы, товары.

---

## Концепция стратегии

### Блок — формирование диапазона

Временной период, в котором определяются границы:
- `range_high` = max(high) за период блока
- `range_low` = min(low) за период блока
- `range_size` = range_high − range_low

Параметры: время начала, время окончания (UTC). Опционально блок может начинаться с предыдущего торгового дня (например, 18:00 вчера — 08:00 сегодня).

### Сессия — торговый период

Период после блока, в котором происходит поиск входа и исполнение одной сделки. Начинается после окончания блока.

### Стартовая позиция

На первой свече сессии определяется положение цены:

| Позиция | Условие                     |
|---------|-----------------------------|
| INSIDE  | range_low ≤ close ≤ range_high |
| ABOVE   | close > range_high          |
| BELOW   | close < range_low           |

### Касание границы

Свеча коснулась уровня, если: `candle.low ≤ level ≤ candle.high`

---

## Типы входов

### Режим по-тренду (Trend)

Вход в направлении пробоя.

| Тип               | Позиция | Условие              | Направление |
|-------------------|---------|----------------------|-------------|
| ENTRY_LONG_TREND  | INSIDE  | Касание range_high   | LONG        |
| ENTRY_SHORT_TREND | INSIDE  | Касание range_low    | SHORT       |
| LIMIT_LONG_TREND  | BELOW   | Возврат к range_low  | LONG        |
| LIMIT_SHORT_TREND | ABOVE   | Возврат к range_high | SHORT       |

### Возвратный режим (Reverse)

Вход в направлении отбоя от границы.

| Тип                  | Позиция | Условие              | Направление |
|----------------------|---------|----------------------|-------------|
| ENTRY_SHORT_REVERSE  | INSIDE  | Касание range_high   | SHORT       |
| ENTRY_LONG_REVERSE   | INSIDE  | Касание range_low    | LONG        |
| LIMIT_SHORT_REVERSE  | ABOVE   | Возврат к range_high | SHORT       |
| LIMIT_LONG_REVERSE   | BELOW   | Возврат к range_low  | LONG        |

### Лимитный вход для ENTRY (опционально)

При включении для ENTRY-типов (цена внутри) требуется дополнительное условие: цена сначала пересекает границу (уходит за неё), затем возвращается и касается. Имитирует лимитный ордер.

### Ситуации без входа

- **INSIDE_BLOCK** — цена не коснулась ни одной границы за сессию
- **OUTSIDE_BLOCK** — цена не вернулась к ближней границе за сессию

## Управление позицией

### Расчёт TP/SL через множители

```
TP_distance = range_size × tp_multiplier
SL_distance = range_size × sl_multiplier
```

LONG: TP = entry + TP_distance, SL = entry − SL_distance
SHORT: TP = entry − TP_distance, SL = entry + SL_distance

Множители настраиваются: 0.1–3.0, шаг 0.1.

### Фиксированные TP/SL

Если range_size выходит за пороги (threshold_min / threshold_max), вместо множителей используются фиксированные расстояния fixed_tp_distance и fixed_sl_distance.

### Симуляция исполнения

После входа система идёт свеча за свечой:
1. Если high/low прошёл через TP → закрытие с прибылью
2. Если high/low прошёл через SL → закрытие с убытком
3. Если новость в буферном окне → закрытие по безубытку (BE)
4. Конец сессии → принудительное закрытие по текущей цене (SESSION_CLOSE)

---

## R-система

Все результаты в R-единицах (1R = размер стоп-лосса).

```
R = (exit − entry) / sl_distance     для LONG
R = (entry − exit) / sl_distance     для SHORT
```

### Коэффициенты

| Параметр                 | Диапазон | Применяется к  | Логика                           |
|--------------------------|----------|----------------|----------------------------------|
| tp_coefficient           | 0.5–1.0  | Только TP      | Уменьшает прибыль (спред/проск.) |
| sl_slippage_coefficient  | 1.0–1.5  | Только SL      | Увеличивает убыток               |
| commission_rate          | 0–1%     | Все сделки     | Round-trip комиссия в R          |

Формула комиссии:
```
commission_R = (entry_price × commission_rate × 2) / sl_distance
R_final = R_base × coefficient − commission_R
```

Пример для Binance Futures: commission_rate = 0.0005 (0.05% за сторону) = 0.10% round-trip. При входе по 2000 и SL = 50 пунктов: commission_R ≈ 0.04R на сделку.

### Max Drawdown

Рассчитывается по кумулятивной серии R: отслеживается пик, на каждом шаге фиксируется просадка = текущее − пик. Максимальная просадка — наибольшее отрицательное значение.

---

## Фильтрация по новостям

### Блокировка входа

Перед входом проверяется окно `[entry_time − buffer, entry_time + buffer]`. Если есть новость выбранной важности по выбранной валюте — вход не происходит.

Фильтр валюты привязывает новости к инструменту: ETHUSDT → USD, EURUSD → EUR + USD.

### Закрытие при новости

Если позиция уже открыта и свеча попадает в буферное окно новости → закрытие по безубытку (BE).

### Пропуск дней

Отдельная опция: полностью исключить дни с high impact новостями.

## Оптимизация

### Стандартная (TP/SL)

Перебор комбинаций tp_multiplier × sl_multiplier в заданных диапазонах. Все остальные настройки (время, режим, фильтры, limit_only_entry) наследуются из сохранённых.

### Цели оптимизации

| Цель            | Формула                              | Когда использовать            |
|-----------------|--------------------------------------|-------------------------------|
| max_total_r     | sum(R)                               | Максимальная прибыль          |
| max_r_dd_ratio  | total_r / abs(max_drawdown)          | Баланс прибыли и просадки     |
| max_r_minus_dd  | total_r − 2 × abs(max_drawdown)      | Жёсткий штраф за просадку     |

`max_r_dd_ratio` (Calmar-like) — ищет золотую середину: высокий R при малой просадке. `max_r_minus_dd` — более консервативный, сильнее штрафует за глубокие просадки.

### Оптимизация времени

Чекбокс «Оптимизация времени разделения Блок/Сессия». Фиксируются начало блока и конец сессии. Перебирается час, в который блок заканчивается и сессия начинается (block_end = session_start). Диапазон: 03:00–18:00, шаг 1 час. Для каждого часа запускается полный перебор TP/SL.

Поддерживает флаг «Блок с предыдущего дня».

Результат: таблица всех временных окон с лучшей метрикой, bar chart, лучшее окно + TP/SL.

### Параллельная обработка

Опциональная многопоточность (ThreadPoolExecutor, до 4 потоков). Кеш результатов защищён threading.Lock для потокобезопасности.

## Отчёты

### Уровни

- **Общий обзор** — Total R, Win Rate, Avg R/trade, Profit Factor, Max Drawdown, график кумулятивного R
- **По годам** — разбивка по каждому году с отдельными графиками
- **По месяцам** — таблица с помесячной статистикой
- **Все сделки** — полная таблица с фильтрами по дате, типу входа, результату, экспорт в CSV

### Метрики

| Метрика          | Описание                                      |
|------------------|-----------------------------------------------|
| Total R          | Сумма всех R                                  |
| Win Rate         | % сделок с TP                                 |
| Avg R per Trade  | Средний R на сделку                           |
| Avg R per Day    | Средний R на торговый день                    |
| Profit Factor    | Сумма прибыльных R / сумма убыточных R        |
| Max Drawdown     | Максимальная просадка кумулятивного R          |
| Best/Worst Trade | Лучшая и худшая сделка в R                    |

## Настройки — сохранение и загрузка

Все настройки сохраняются в JSON: время блока/сессии, режим, множители, коэффициенты, фильтры, период. При сохранении показывается expander со всеми параметрами для проверки.

Загрузка JSON: файл читается, значения применяются через промежуточный буфер (`_pending_settings`) перед рендером виджетов — это решает проблему Streamlit с изменением session_state после создания виджета.
